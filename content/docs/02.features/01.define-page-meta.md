## 概述

`definePageMeta` 是 uni-pages 提供的编译时宏，用于在 Vue 组件中直接定义页面的配置信息。这些配置会在构建时被提取并合并到 `pages.json` 中。

```vue
<script setup lang="ts">
definePageMeta({
  type: 'home',
  style: {
    navigationBarTitleText: '首页',
  },
})
</script>
```

::tip
`definePageMeta` 不需要导入，它会在构建时被自动识别和处理。
::

## 基础用法

### 标记首页

使用 `type: 'home'` 将页面标记为应用启动页：

```vue
<script setup lang="ts">
definePageMeta({
  type: 'home',
  style: {
    navigationBarTitleText: '首页',
  },
})
</script>
```


### 配置导航栏

```vue
<script setup lang="ts">
definePageMeta({
  style: {
    navigationBarTitleText: '我的页面',
    navigationBarBackgroundColor: '#333333',
    navigationBarTextStyle: 'white',
  },
})
</script>
```

## 配置属性

### type

- **类型**：`'home' | 'page'`
- **默认值**：`'page'`

页面类型。设为 `'home'` 时，该页面会被放在 `pages.json` 的 pages 数组首位。

### style

页面窗口样式配置，与 UniApp `pages.json` 中的 `style` 属性一致。

## 条件编译

uni-pages 支持在 `definePageMeta` 中使用平台条件编译语法，可以为不同平台设置不同的配置值：

```vue
<script setup lang="ts">
definePageMeta({
  style: {
    navigationBarTitleText: {
      'h5': 'H5 标题',
      'mp-weixin': '微信小程序标题',
      'default': '默认标题',
    },
  },
})
</script>
```

::callout{icon="i-lucide-book-open" color="primary"}
完整的条件编译语法和示例请参阅 [条件编译](/pages/docs/features/conditional-compilation) 文档。
::

## 注意事项

::warning
**编译时限制**：`definePageMeta` 中只能使用静态值，不能使用变量或函数调用。
::

```vue
<script setup lang="ts">
// ❌ 错误：不能使用变量
const title = '动态标题'
definePageMeta({
  style: {
    navigationBarTitleText: title,  // 这不会工作
  },
})

// ✅ 正确：使用静态字符串
definePageMeta({
  style: {
    navigationBarTitleText: '静态标题',
  },
})
</script>
```
